# Contributing to StartQ

First, thank you for considering contributing to StartQ! 

StartQ is designed to be the foundational operational layer for AI agents. Our goal is strict stability and zero amnesia. 

## How to Contribute

### 1. Report Bugs
Use the GitHub issue tracker to report bugs. Please ensure you fill out the provided Bug Report template completely. If your agent is hallucinating state, please include the exact sequence of `startq` commands executed.

### 2. Suggest Enhancements
If you have an idea for a new feature (e.g. supporting a new database driver like PostgreSQL for the Brain), please submit a Feature Request issue *before* writing code. We want to ensure it aligns with the Phase 1 scope.

### 3. Pull Requests
- Fork the repository.
- Ensure your code passes all lint checks (`flake8`, `black`).
- Ensure unit tests pass locally before submitting.
- Provide a clear, detailed PR description. Do not bundle multiple unrelated features into a single PR.

Thank you for helping us make AI architectures reliable!
